# Plan 1.1: Setup Migration Script and Test Parsing

## Objective
Write a Node.js script that can recursively parse all MDX files in `classic/docs/`, extract their frontmatter correctly, and prepare the content for Sanity CMS. Verify the script runs correctly locally against `getting-started/`.

## Context
- .gsd/SPEC.md
- classic/scripts/fetch-sanity-content.js

## Tasks

<task type="auto">
  <name>Create basic script structure</name>
  <files>classic/scripts/migrate-to-sanity.js</files>
  <action>
    Create a new Node script in `classic/scripts/` that can find all `.md` and `.mdx` files in `classic/docs/`.
    - It should parse yaml frontmatter (e.g., using a regex or installing `gray-matter`).
    - It must convert the text to basic Sanity portable text blocks. 
    - The schema type mapping (`doc`, `releaseNote`, `article`, etc) should be derived from the folder path (e.g. `classic/src/pages/releases/` -> `releaseNote`, otherwise `doc`).
  </action>
  <verify>Run the script in dry-run mode and log out the mapped slugs and titles to ensure they match exactly.</verify>
  <done>Script logs output showing correctly extracted titles, slugs, and mapped types.</done>
</task>
