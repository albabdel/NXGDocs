  <name>task 1: Use light-dark() for theme tokens</name>
  <files>classic/src/css/tokens.css</files>
  <action>
    Replace `:root + [data-theme='dark']` pairs with `light-dark()` where applicable.
    
    Browser support: 93%+ (Chrome 123+, Firefox 120+, Safari 17.5+). Safe for a docs site.
    
    Add at the top of tokens.css in :root:
    ```css
    :root {
      /* Enable color-scheme for light-dark() */
      color-scheme: light dark;
      
      /* === GLASSMORPHISM TOKENS - Using light-dark() === */
      --glass-bg: light-dark(rgba(255,255,255,0.72), rgba(255,255,255,0.025));
      --glass-bg-hover: light-dark(rgba(255,255,255,0.85), rgba(255,255,255,0.05));
      --glass-border: light-dark(rgba(232,176,88,0.12), rgba(255,255,255,0.07));
      --glass-border-hover: light-dark(rgba(232,176,88,0.2), rgba(255,255,255,0.12));
      
      /* === ICON TOKENS - Using light-dark() === */
      --icon-bg: light-dark(rgba(232,176,88,0.10), rgba(232,176,88,0.12));
      --icon-bg-hover: light-dark(rgba(232,176,88,0.15), rgba(232,176,88,0.18));
      
      /* === BORDER TOKENS - Using light-dark() === */
      --border-subtle: light-dark(rgba(0,0,0,0.1), rgba(255,255,255,0.06));
      --border-default: light-dark(#E5E7EB, rgba(255,255,255,0.12));
      
      /* === SHADOW TOKENS - Using light-dark() === */
      --shadow-sm: light-dark(
        0 1px 2px rgba(0,0,0,0.06), 0 1px 3px rgba(0,0,0,0.10),
        0 1px 2px rgba(0,0,0,0.30), 0 1px 3px rgba(0,0,0,0.40)
      );
    }
    ```
    
    Note: Keep the `[data-theme='dark']` block for tokens that need more complex values or for backwards compatibility during transition.
    
    The `light-dark()` function automatically switches based on the computed `color-scheme` property, which Docusaurus sets via `[data-theme]` attribute.

<task type="auto">
  <name>task 2: Add container queries support</name>
  <files>classic/src/css/components/cards.css</files>
  <action>
    Add container queries for responsive card layouts.
    
    Add to cards.css:
    ```css
    /* Container Query Support for Cards */
    .card-container {
      container-type: inline-size;
      container-name: card;
    }
    
    /* Responsive card layout based on container width */
    @container card (min-width: 480px) {
      .card-horizontal {
        display: grid;
        grid-template-columns: auto 1fr;
        gap: 1rem;
      }
      
      .card-horizontal .card-icon {
        grid-row: span 2;
      }
    }
    
    @container card (max-width: 479px) {
      .card-horizontal {
        display: flex;
        flex-direction: column;
      }
      
      .card-horizontal .card-icon {
        margin-bottom: 0.5rem;
      }
    }
    
    /* Feature card container */
    .feature-card-container {
      container-type: inline-size;
    }
    
    @container (min-width: 320px) {
      .feature-card {
        padding: 1.5rem;
      }
    }
    
    @container (min-width: 480px) {
      .feature-card {
        padding: 2rem;
      }
    }
    ```
    
    This allows cards to respond to their container size rather than viewport size, enabling more flexible layouts.
  </action>
