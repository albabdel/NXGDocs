<task type="auto">
  <name>Create Confluence Surge import script</name>
  <files>
    - scripts/import-confluence-surge.ts
  </files>
  <action>
    Create the main import script at `scripts/import-confluence-surge.ts` that:
    
    1. **Environment Setup**
       - Reads CONFLUENCE_EMAIL, CONFLUENCE_API_TOKEN, CONFLUENCE_SITE_URL from env
       - Reads SANITY_PROJECT_ID (fjjuacab), SANITY_DATASET (production), SANITY_TOKEN from env
       - Validates all required environment variables before proceeding
    
    2. **Content Fetching**
       - Uses `fetchSpaceContent()` from `functions/lib/confluence-service.ts`
       - Fetches all pages from Surge space (limit: 500, with pagination if needed)
       - Filters for status='current' (published pages only)
    
    3. **Content Transformation**
       - Converts Confluence storage format to Sanity Portable Text
       - Extracts title from page
       - Generates slug from title (lowercase, hyphenated)
       - Sets product='gcsurge'
       - Sets status='published' (or 'draft' if Confluence status is draft)
       - Preserves page hierarchy (parent-child relationships)
    
    4. **Sanity Document Creation**
       - Creates doc documents in Sanity
       - Uses transaction for batch inserts
       - Logs success/failure for each page
       - Creates mapping of Confluence page ID to Sanity document ID
    
    5. **Error Handling**
       - Graceful handling of API errors
       - Skip pages that fail transformation
       - Continue with remaining pages
       - Summary report at end
    
    Key imports:
    ```typescript
    import { fetchSpaceContent, fetchSpaceInfo, type ConfluencePage } from '../functions/lib/confluence-service';
    import { createClient } from '@sanity/client';
    import { CONFLUENCE_SURGE_CONFIG } from './import-confluence-surge-config';
    ```
    
    The script should be executable via: `npx ts-node scripts/import-confluence-surge.ts`
  </action>
