    Create `classic/src/components/UpdateCard.tsx` with type-specific card variants:

    1. Define shared UpdateCard props interface:
       - `update: Update` (from updates.tsx type)
       - Optional `className` for styling

    2. Create UpdateCard component with switch on `update.type`:
       ```tsx
       export function UpdateCard({ update }: UpdateCardProps) {
         switch (update.type) {
           case 'release': return <ReleaseCard update={update} />
           case 'bugfix': return <BugfixCard update={update} />
           case 'roadmap': return <RoadmapCard update={update} />
           case 'announcement': return <AnnouncementCard update={update} />
         }
       }
       ```

    3. ReleaseCard (type='release'):
       - Version badge (prominent, gold/blue color)
       - Title
       - Excerpt
       - Sections preview: "X new, Y improvements, Z fixes" (sum from releaseNotes arrays)
       - Published date
       - Link to /updates/[slug]
       - Style: Full card with version prominently displayed

    4. BugfixCard (type='bugfix'):
       - Title only (minimal)
       - Status badge (fixed = green, monitoring = yellow)
       - Severity indicator if present (colored dot)
       - Published date
       - Link to /updates/[slug]
       - Style: Compact, minimal, NOT blog-style
       - UX Rule: Keep bug fixes SHORT

    5. RoadmapCard (type='roadmap'):
       - Title
       - Excerpt
       - Status badge (planned = gray, in_progress = blue, completed = green)
       - Target date if present
       - Link to /updates/[slug]
       - Style: Progress tracking feel

    6. AnnouncementCard (type='announcement'):
       - Title
       - Excerpt (blog-style preview)
       - Published date
       - Link to /updates/[slug]
       - Style: Blog-style title + excerpt
