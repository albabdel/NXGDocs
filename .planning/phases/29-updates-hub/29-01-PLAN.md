<task type="auto">
  <name>Create update schema with type enum and conditional fields</name>
  <files>studio/schemaTypes/update.ts</files>
  <action>
    Create `studio/schemaTypes/update.ts` with the following structure:

    1. Define `updateType` document schema with:
       - `name: 'update'`
       - `title: 'Update'`
       - Preview configuration showing title + type badge

    2. Add field groups:
       - `main` — title, type, publishedAt, slug
       - `content` — excerpt, content (portable text)
       - `releaseFields` — conditional, for type='release'
       - `bugfixFields` — conditional, for type='bugfix'
       - `roadmapFields` — conditional, for type='roadmap'

    3. Core fields (group: main):
       - `title` (string, required)
       - `slug` (slug, source from title)
       - `type` (string, options: announcement/release/bugfix/roadmap, required)
       - `publishedAt` (datetime)

    4. Content fields (group: content):
       - `excerpt` (text, rows: 2, description: "Short summary for card display")
       - `content` (portable text / array)

    5. Release-specific fields (group: releaseFields, hidden unless type='release'):
       - `version` (string, description: "e.g., v3.2.1")
       - `releaseNotes` (object):
         - `new` (array of objects: title, description)
         - `improvements` (array of objects: title, description)
         - `fixes` (array of objects: title, description)

    6. Bugfix-specific fields (group: bugfixFields, hidden unless type='bugfix'):
       - `status` (string, options: fixed/monitoring)
       - `severity` (string, options: low/medium/high, optional)

    7. Roadmap-specific fields (group: roadmapFields, hidden unless type='roadmap'):
       - `roadmapStatus` (string, options: planned/in_progress/completed)
       - `targetDate` (date, optional)

    Use `hidden: ({parent}) => parent?.type !== 'release'` pattern for conditional fields.

    Follow schema patterns from `landingPage.ts` for:
    - defineType/defineField imports
    - Preview configuration with emoji badges
    - Field group syntax with `options: { collapsible: true }`

    Do NOT:
    - Create separate schemas per type (single update schema only)
    - Add tags field (deferred)
    - Add relations to other documents (deferred)
  </action>
  <verify>

  </verify>

    - Conditional fields hidden until matching type selected
    - Field groups configured (Main, Content, Release, Bugfix, Roadmap)

  </done>
