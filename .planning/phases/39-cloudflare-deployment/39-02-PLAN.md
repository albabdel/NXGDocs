  <name>task 1: Create product detection utility</name>
  <files>functions/lib/webhook-router.ts</files>
  <action>
Create a utility module that detects which product(s) a Sanity document belongs to.

The module should export:

1. `detectProductFromPayload(payload: SanityWebhookPayload): string[]`
   - Extracts the `product` field from the document in the webhook payload
   - Returns array of products to rebuild (single product or multiple for shared content)
   - Handles cases where product field might be missing (default to ['gcxone'])
   - For `product: 'shared'` documents, returns all products ['gcxone', 'gcsurge']

2. `routeToProduct(products: string[]): ProductBuildConfig[]`
   - Maps product names to their Cloudflare Pages deployment hook URLs
   - Returns configuration for each product that needs rebuilding

3. `validateSanitySignature(request: Request, secret: string): boolean`
   - Validates the X-Sanity-Signature header
   - Prevents unauthorized webhook calls

Type definitions:
```typescript
interface SanityWebhookPayload {
  _id: string;
  _type: string;
  product?: string;
  slug?: { current: string };
  // ... other Sanity document fields
}

interface ProductBuildConfig {
  product: string;
  projectName: string;
  deploymentHookUrl: string;
}
```

Product mapping:
- 'gcxone' -> gcxone-docs project
- 'gcsurge' -> gcsurge-docs project
- 'shared' -> all projects
- undefined/missing -> ['gcxone'] (backwards compatibility)

<task type="auto">
  <name>task 2: Create Sanity webhook handler Function</name>
  <files>functions/sanity-webhook.ts</files>
  <action>
Create a Cloudflare Pages Function that receives Sanity webhooks and triggers the appropriate product build(s).

The function should:

1. Receive POST requests at /api/sanity-webhook
2. Validate Sanity signature using the secret (read from SANITY_WEBHOOK_SECRET env var)
3. Parse the webhook payload to extract document info
4. Detect which product(s) the document belongs to using detectProductFromPayload()
5. For each affected product:
   - Call Cloudflare Pages deployment hook API
   - POST to dispatch deployment endpoint
6. Return appropriate response:
   - 200 OK with rebuild status
   - 401 Unauthorized if signature invalid
   - 500 Internal Server Error if deployment trigger fails

Cloudflare Pages API for triggering deployments:
```
POST https://api.cloudflare.com/client/v4/accounts/{account_id}/pages/projects/{project_name}/deployments
Authorization: Bearer {api_token}
Content-Type: application/json

{
  "branch": "main"
}
```

Alternative: Use deployment hooks (simpler):
Each Pages project has a unique deployment hook URL:
```
POST https://api.cloudflare.com/client/v4/accounts/{account_id}/pages/projects/{project}/deployments
```

Include detailed logging for debugging:
- Which product(s) detected
- Which builds triggered
- Success/failure status
  </action>
