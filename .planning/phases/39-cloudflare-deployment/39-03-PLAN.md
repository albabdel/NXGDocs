<task type="auto">
  <name>task 2: Create deployment runbook</name>
  <files>docs/deployment-runbook.md</files>
  <action>
Create a comprehensive deployment runbook documenting the multi-product deployment process.

The runbook should include:

## 1. Architecture Overview
- Two Cloudflare Pages projects: gcxone-docs, gcsurge-docs
- Custom domains: docs.gcxone.com, docs.gcsurge.com
- Sanity webhook triggers product-scoped rebuilds

## 2. Deployment Flow
```
Sanity publish → Webhook → Product detection → Trigger Cloudflare build
```

## 3. Manual Deployment Procedures

### Deploy Single Product
```bash
# Build specific product
PRODUCT=gcxone npm run build

# Deploy to Cloudflare
npx wrangler pages deploy classic/build/gcxone --project-name=gcxone-docs
```

### Deploy All Products
```bash
node scripts/deploy-multi-product.js
```

### Trigger Build via API
```bash
node scripts/trigger-product-build.js --product gcxone
```

## 4. Adding a New Product

Step-by-step process:
1. Add product to product.config.ts
2. Add product enum to Sanity schemas
3. Create Cloudflare Pages project
4. Configure custom domain
5. Set environment variables (PRODUCT=newproduct)
6. Update webhook router to handle new product
7. Create initial content in Sanity

## 5. Troubleshooting

### Build fails
- Check Cloudflare Pages logs
- Verify SANITY_API_TOKEN is valid
- Check PRODUCT env var is set

### Wrong content appears on domain
- Verify PRODUCT env var in Cloudflare project
- Check GROQ product filter in fetch-sanity-content.js
- Run verify-deployment.js script

### Webhook not triggering
- Check SANITY_WEBHOOK_SECRET matches
- Verify webhook URL in Sanity dashboard
- Check Cloudflare Function logs

## 6. Environment Variables Reference

| Variable | gcxone-docs | gcsurge-docs |
|----------|-------------|--------------|
| PRODUCT | gcxone | gcsurge |
| SANITY_PROJECT_ID | fjjuacab | fjjuacab |
| SANITY_DATASET | production | production |
| SANITY_API_TOKEN | (secret) | (secret) |
| SANITY_WEBHOOK_SECRET | (secret) | (secret) |

## 7. Rollback Procedure

```bash
# List recent deployments
npx wrangler pages deployment list --project-name=gcxone-docs

# Rollback to previous deployment
npx wrangler pages deployment rollback --project-name=gcxone-docs
```
