  <name>task 1: Create shared session utilities</name>
  <files>
    functions/lib/auth-session.ts
  </files>
  <action>
    Create `functions/lib/auth-session.ts` with the following utilities:

    1. **Types:**
       ```typescript
       export interface AuthSession {
         userId: string;        // Auth0 'sub' claim
         email: string;
         name: string;
         picture?: string;
         role: 'user' | 'admin' | 'operator' | 'manager';
         orgId?: string;
         iat: number;
         exp: number;
       }
       
       export interface Env {
         AUTH0_DOMAIN: string;
         SESSION_SECRET: string;
       }
       ```

    2. **createSession function:**
       - Takes user claims and session secret
       - Creates HMAC-SHA256 signed session token
       - Returns session token string
       - Session expires in 24 hours
       - Pattern matches existing `lib/zoho-session.ts` but for Auth0

    3. **verifySession function:**
       - Takes session token and secret
       - Verifies HMAC signature
       - Checks expiration
       - Returns parsed session or null

    4. **buildSessionCookie function:**
       - Takes session token
       - Returns Set-Cookie header value
       - Cookie attributes: HttpOnly, Secure, SameSite=Lax, Path=/
       - Max-Age: 86400 (24 hours)

    5. **getSessionFromHeader function:**
       - Extracts session cookie from Cookie header
       - Verifies and parses session
       - Returns AuthSession or null

    Reference the existing patterns in:
    - `functions/lib/zoho-session.ts` (90% similar, adapt for Auth0)
    - `functions/lib/admin-session.ts`
  </action>

<task type="auto">
  <name>task 2: Create session validation endpoint</name>
  <files>
    functions/auth-session.ts
  </files>
  <action>
    Create `functions/auth-session.ts`:

    1. **onRequestGet handler:**
       - Check for existing session cookie first
       - If session valid, return user profile
       - If no session, check for Auth0 ID token in Authorization header
       - Validate Auth0 ID token using JWKS (reuse from zoho-customer-auth.ts)
       - Create new session from Auth0 claims
       - Return user profile with Set-Cookie header

    2. **Response format:**
       ```typescript
       // Authenticated response
       {
         authenticated: true,
         user: {
           userId: string,
           email: string,
           name: string,
           picture?: string,
           role: string
         }
       }
       
       // Not authenticated response
       {
         authenticated: false
       }
       ```

    3. **Security considerations:**
       - Always use HTTPS (enforced by Cloudflare)
       - HttpOnly cookie (JavaScript cannot access)
       - SameSite=Lax (CSRF protection)
       - Session secret from environment variable

    4. **JWKS verification:**
       - Reuse `getSigningKey` function from zoho-customer-auth.ts
       - Cache JWKS for 1 hour (use Cloudflare cache)
       - Validate RS256 signature

    Reference:
    - `functions/zoho-customer-auth.ts` for JWKS verification pattern
    - `functions/lib/zoho-session.ts` for session creation pattern
  </action>
