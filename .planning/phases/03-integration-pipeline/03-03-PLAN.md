    This task requires dashboard interaction in two external services. Claude cannot automate
    OAuth-authenticated dashboard actions. Follow these exact steps:

    STEP 1 — Create Cloudflare Pages deploy hook:
    1. Log in to https://dash.cloudflare.com
    2. Navigate: Workers and Pages → select the "nxgen" project
    3. Click Settings → Builds tab
    4. Scroll to "Deploy hooks" section → click "Add deploy hook"
    5. Name: sanity-publish
    6. Branch to build: main
    7. Click "Save" — Cloudflare generates a unique hook URL
    8. Copy the hook URL — it looks like:
       https://api.cloudflare.com/client/v4/pages/webhooks/deploy_hooks/[uuid]
    9. Keep this URL — you need it in Step 2. Do NOT commit it to the repo.

    STEP 2 — Test the deploy hook (confirm it works before wiring Sanity):
    From your local terminal (with the URL from Step 1):

      curl -X POST "https://api.cloudflare.com/client/v4/pages/webhooks/deploy_hooks/[paste-your-uuid-here]"

    Expected: Response body contains {"result":{"id":"...","url":"..."}} with HTTP 200.
    In CF Pages dashboard, a new deployment entry should appear within ~30 seconds.

    STEP 3 — Register Sanity webhook:
    1. Go to https://manage.sanity.io
    2. Select the nxgen-docs project
    3. Click API → Webhooks → "Add webhook"
    4. Fill in:
       - Name: cloudflare-pages-rebuild
       - URL: [paste the CF deploy hook URL from Step 1]
       - Trigger on: Create, Update, Delete (check all three)
       - Filter (GROQ): _type in ["doc", "releaseNote", "article", "referencePage"]
       - Projection: leave empty
       - Secret: leave empty (Phase 3 — v2 concern)
       - Status: Active
       - HTTP method: POST
       - Dataset: production
    5. Click "Save"

    STEP 4 — Confirm webhook appears as Active in the Sanity webhooks list.
