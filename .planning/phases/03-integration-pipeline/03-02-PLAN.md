  <name>Task 1: Create docusaurus-plugin-sanity-content/index.js</name>
  <files>classic/plugins/docusaurus-plugin-sanity-content/index.js</files>
  <action>
    Create the directory `classic/plugins/docusaurus-plugin-sanity-content/` and write `index.js` as a CommonJS module (matching the existing plugin pattern). Do NOT use ES modules or .mjs — the existing plugin uses CJS and so must this one.

    The plugin must:
    1. Validate env vars (SANITY_PROJECT_ID, SANITY_API_TOKEN) — throw a clear error if missing
    2. Create all 6 .sanity-cache/ subdirectories with fs.mkdirSync({ recursive: true }) before any fetch
    3. Run four separate GROQ queries (one per content type) — hard-fail on any Sanity API error
    4. For each document: serialize Portable Text body to MDX, generate frontmatter, write the .mdx file
    5. Apply the audience→directory mapping for doc type; use 'docs' directory for the other three types
    6. Return null from loadContent() — files on disk are the deliverable, not the return value

    Critical implementation notes:
    - Use `@sanity/client` import style: `const { createClient } = require('@sanity/client')`
    - Use `@portabletext/markdown` import style: `const { toMarkdown } = require('@portabletext/markdown')`
      NOTE: verify the actual export name by checking: `Object.keys(require('@portabletext/markdown'))` — the export may be `toMarkdown` or `portableTextToMarkdown`; use whatever the package actually exports
    - Hard-code `apiVersion: '2025-02-06'` — do NOT use `new Date().toISOString()`
    - Set `useCdn: false` — bypass CDN for fresh build-time data
    - Callout renderer: `value.body` is already a plain string — do NOT call portableTextToMarkdown on it
    - Code block renderer: read `value.code` and `value.language` from the block
    - Image renderer: for Phase 3, use `@sanity/image-url` to build the CDN URL from `asset._ref`.
      Install it: `npm install @sanity/image-url` in classic/ if not already present.
      Use: `imageUrlBuilder(client).image(value).url()`
    - Slug: always use `doc.slug.current` for file paths and GROQ filter `defined(slug.current)`
    - File path convention: `{cacheRoot}/{subDir}/{doc.slug.current}.mdx`
    - If a document has no body, write just the frontmatter (no body section)
    - Do NOT throw on a per-document serialization error — log a warning and continue to the next document.
      Only throw on Sanity API fetch errors (the GROQ query itself failing).

    GROQ queries to use (field projections match schema above):
    ```
    *[_type == "doc" && defined(slug.current)] | order(sidebarPosition asc) {
      title, slug, targetAudience, category, sidebarPosition, body, lastUpdated
    }
    *[_type == "releaseNote" && defined(slug.current)] | order(publishedAt desc) {
      title, slug, sprintId, publishedAt, body
    }
    *[_type == "article" && defined(slug.current)] | order(publishedAt desc) {
      title, slug, tags, publishedAt, body
    }
    *[_type == "referencePage" && defined(slug.current)] {
      title, slug, body
    }
    ```

    Frontmatter rules per type:
    - All types: `title` (required, escape double quotes as \")
    - doc: + `sidebar_position` (if sidebarPosition set), `sidebar_label` (if category set), `last_update.date` (if lastUpdated set)
    - releaseNote: + `date` (if publishedAt set), `custom_edit_url: null`
    - article: + `date` (if publishedAt set), `custom_edit_url: null`
    - referencePage: no extra frontmatter beyond title

    Audience routing for doc type:
    - Use `doc.targetAudience` array if present and non-empty; default to `['all']` if absent
    - Write the file once per audience value (multi-audience docs get written to multiple dirs)
    - releaseNote, article, referencePage always go to `.sanity-cache/docs/` only

    Anti-patterns to avoid (from research):
    - Do NOT use `useCdn: true`
    - Do NOT call portableTextToMarkdown on callout.body (it is a plain string)
    - Do NOT use `_type == 'reference'` — use `_type == 'referencePage'`
    - Do NOT place this plugin AFTER plugin-content-docs in docusaurus.config.ts (handled in Task 2)
  </action>
