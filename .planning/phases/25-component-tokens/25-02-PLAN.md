<task type="auto">
  <name>task 2: Migrate TicketDetail inline styles to CSS classes</name>
  <files>classic/src/components/ZohoTickets/TicketDetail.tsx</files>
  <action>
    Migrate TicketDetail.tsx from inline style branches to CSS classes.
    
    This component has 200+ inline style branches. The migration strategy:
    
    1. **Replace style objects with className**:
       - `style={{ background: isDark ? '...' : '...' }}` → `className="ticket-detail-card"`
    
    2. **Pattern to find**:
       ```tsx
       style={{
         background: isDark ? 'rgba(255,255,255,0.06)' : 'rgba(0,0,0,0.05)',
         border: `1px solid ${isDark ? 'rgba(255,255,255,0.1)' : 'rgba(0,0,0,0.1)'}`
       }}
       ```
    
    3. **Pattern to replace**:
       ```tsx
       className="ticket-comment"
       ```
    
    4. **Specific migrations**:
       - CopyButton: Remove isDark prop, use `.ticket-button` class
       - Avatar: Remove isDark prop, use `.ticket-avatar` class
       - TranslateButton: Remove isDark prop, use `.ticket-button` class
       - AttachmentItem: Remove isDark prop, use `.ticket-attachment` class
       - SLATimer: Remove isDark prop, use `.sla-timer` class
       - RelatedArticles: Remove isDark prop, use `.related-article` class
       - StyledSelect: Remove isDark prop, use `.ticket-input` class
    
    5. **Keep isDark only where necessary**:
       - If a component truly needs to know the theme for logic (not styling), keep the prop
       - Most components should NOT need isDark after migration
    
    6. **Test dark/light mode**:
       - Run build to verify no TypeScript errors
       - Visual inspection should show consistent styling
  </action>
