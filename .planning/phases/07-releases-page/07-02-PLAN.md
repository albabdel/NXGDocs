# Plan 7.2: Release Detail Page

## Objective

Create a dynamic route at `/releases/[slug]` that displays the full details of a sprint release, including all items with their titles, descriptions, change-type tags, affected-areas tags, screenshots, video embeds, and optional "Read the docs" links.

## Context

- `.planning/phases/07-releases-page/07-RESEARCH.md` — Pattern 2: Dynamic route
- `.planning/phases/06-schema-and-data-pipeline/06-RESEARCH.md` — JSON item structure
- `classic/src/legacy-pages/releases/sprint-2025-12-b.tsx` — Detail page UI patterns
- `classic/src/data/sanity-releases.generated.json` — Data source

## Prerequisite

Plan 7.1 complete. Release cards link to `/releases/[slug]`.

       - Map over `release.items`

<task type="auto">
  <name>Add screenshots and video embeds to items</name>
  <files>
    - classic/src/pages/releases/[slug].tsx
  </files>
  <action>
    Enhance each item in the items list:
    
    1. Screenshot: If `item.screenshotUrl` exists, render `<img>`:
       ```tsx
       {item.screenshotUrl && (
         <img 
           src={item.screenshotUrl} 
           alt={item.title}
           className="rounded-lg border border-white/10 my-4"
         />
       )}
       ```
    
    2. Video embed: If `item.videoUrl` exists, extract embed URL and render iframe:
       ```tsx
       function getVideoEmbedUrl(url: string): string | null {
         // YouTube
         const ytMatch = url.match(/(?:youtube\.com\/watch\?v=|youtu\.be\/)([^&]+)/);
         if (ytMatch) return `https://www.youtube.com/embed/${ytMatch[1]}`;
         // Vimeo
         const vimeoMatch = url.match(/vimeo\.com\/(\d+)/);
         if (vimeoMatch) return `https://player.vimeo.com/video/${vimeoMatch[1]}`;
         return null;
       }
       
       {item.videoUrl && getVideoEmbedUrl(item.videoUrl) && (
         <iframe 
           src={getVideoEmbedUrl(item.videoUrl)}
           className="w-full aspect-video rounded-lg my-4"
           allowFullScreen
         />
       )}
       ```
    
    3. Responsive styling: Use existing Tailwind/utility classes for responsive images and videos
    
    Do NOT:
    - Use @sanity/client for image resolution (URL already in JSON)
    - Import heavy video player libraries (native iframe is sufficient)
  </action>
  <verify>Visual: items with screenshots/videos render correctly</verify>
  <done>
    - Screenshots display inline with correct aspect ratio
    - YouTube/Vimeo videos embed and play
    - Responsive layout works on mobile
  </done>
</task>
