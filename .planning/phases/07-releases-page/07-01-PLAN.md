# Plan 7.1: Releases Index Page

## Objective

Replace the `SanityLandingPageRoute` wrapper in `releases.tsx` with a full React page that imports `sanity-releases.generated.json` directly and renders a reverse-chronological list of sprint releases with cards showing title, date, summary, and change-type badges. The most recent release displays a "Latest" badge.

## Context

- `.planning/phases/06-schema-and-data-pipeline/06-RESEARCH.md` — JSON structure
- `.planning/research/ARCHITECTURE.md` — Pattern 1: Direct JSON import
- `classic/src/legacy-pages/releases.tsx` — UI patterns to adapt
- `classic/src/data/sanity-releases.generated.json` — Data source (Phase 6 output)

## Prerequisite

Phase 6 must produce `sanity-releases.generated.json` with mock data (MOCK-01). If JSON is empty, page shows empty state.
