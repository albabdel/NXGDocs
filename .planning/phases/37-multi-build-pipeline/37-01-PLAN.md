  <action>
Create `classic/scripts/build-multi-product.js` that orchestrates builds for each product. This implements ARCHITECTURE.md Pattern 1 (Environment-Driven Product Context) and the multi-build pipeline flow diagram.

The script should:

1. Define PRODUCTS array at top:
```javascript
const PRODUCTS = ['gcxone', 'gcsurge'];
```

2. Create main `buildAllProducts()` function that:
   - Accepts optional `products` array argument (defaults to PRODUCTS)
   - Accepts optional `parallel` boolean argument (default: true for 2 products)
   - Logs start time and product list

3. Create `buildProduct(product)` async function that:
   - Sets `process.env.PRODUCT = product`
   - Calls `fetch-sanity-content.js` run() with PRODUCT env set
   - Calls `generate-sidebar-from-sanity.js` (via require) with PRODUCT env set
   - Spawns `npx docusaurus build --out-dir build/${product}` with inherited stdio
   - Returns build result with timing

4. Implement parallel builds using Promise.all() for 2 products:
```javascript
async function buildAllProducts(products = PRODUCTS, parallel = true) {
  const startTime = Date.now();
  console.log(`[multi-build] Starting builds for: ${products.join(', ')}`);
  
  if (parallel) {
    const results = await Promise.all(products.map(buildProduct));
    // Log results
  } else {
    for (const product of products) {
      await buildProduct(product);
    }
  }
  
  const duration = ((Date.now() - startTime) / 1000).toFixed(1);
  console.log(`[multi-build] All builds complete in ${duration}s`);
}
```

5. Handle errors gracefully:
   - Catch build failures and log which product failed
   - Continue with remaining products in parallel mode
   - Exit with error code if any build fails

6. Export for programmatic use:
```javascript
module.exports = { buildAllProducts, buildProduct, PRODUCTS };
```

7. CLI entry point:
```javascript
if (require.main === module) {
  const args = process.argv.slice(2);
  const products = args.length > 0 ? args : PRODUCTS;
  const parallel = !args.includes('--sequential');
  buildAllProducts(products, parallel).catch(err => {
    console.error(`[multi-build] Error: ${err.message}`);
    process.exit(1);
  });
}
```

CRITICAL: The script must set `process.env.PRODUCT` BEFORE requiring `fetch-sanity-content.js` because that script reads the env var at module load time. Use `delete require.cache[require.resolve('./fetch-sanity-content')]` between builds to ensure fresh module load with new PRODUCT value.

IMPORTANT: The Docusaurus `--out-dir` flag specifies the build output directory. Each product gets its own: `build/gcxone/`, `build/gcsurge/`.
