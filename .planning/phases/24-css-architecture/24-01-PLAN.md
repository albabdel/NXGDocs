<task type="auto">
  <name>task 1: Add CSS @layer architecture to custom.css</name>
  <files>classic/src/css/custom.css</files>
  <action>
    At the very top of custom.css (after the @tailwind directives), add CSS @layer declarations:
    
    ```css
    /* CSS Layer Architecture - Override Docusaurus without !important */
    @layer docusaurus, tokens, components, utilities, overrides;
    ```
    
    Then wrap the existing CSS blocks in appropriate layers:
    
    1. `@layer tokens { ... }` — Wrap all `:root` variable definitions and `[data-theme='dark']` blocks
    2. `@layer components { ... }` — Wrap all component-specific CSS (cards, buttons, sidebar, navbar, tables, code blocks)
    3. `@layer utilities { ... }` — Wrap utility classes
    4. `@layer overrides { ... }` — Wrap any Docusaurus overrides
    
    The @layer order declaration at the top establishes precedence: later layers override earlier ones, and unlayered Docusaurus styles lose to all layered styles.
    
    DO NOT change any actual style values — only add the @layer wrapper structure.
  </action>
  <verify>
    <automated>grep -c "@layer" classic/src/css/custom.css</automated>
    <manual>Verify @layer count is at least 5 (declaration + 4 wrapped blocks)</manual>
    <sampling_rate>run after task commits</sampling_rate>
  </verify>
  <done>custom.css has @layer declaration at top and existing styles wrapped in appropriate layers</done>
</task>
