<!-- Key env var naming convention from research — critical to get right -->

Studio-side (used by sanity.config.ts / sanity.cli.ts — browser bundle):
  SANITY_STUDIO_PROJECT_ID=<project-id>
  SANITY_STUDIO_DATASET=production

Cloudflare Pages / Phase 3 plugin (Node.js, not browser):
  SANITY_PROJECT_ID=<project-id>      ← note: no STUDIO_ prefix
  SANITY_DATASET=production
  SANITY_API_TOKEN=<editor-token>

These are two DIFFERENT env var sets with different names for different consumers.
Do NOT set SANITY_STUDIO_* in Cloudflare Pages — those are local-only for Studio dev/deploy.

**Step 1 — Confirm Studio is accessible:**
Navigate to https://nxgen-docs.sanity.studio (or the alternate URL from the deploy output).
Sign in with your Sanity account. Expected: Studio loads and shows the NXGEN Docs workspace.

**Step 2 — Verify all 4 document types appear in sidebar:**
In the Studio left sidebar, you should see: Documentation Page, Release Note, Article, Reference Page.
If any are missing, the schema index.ts export is incomplete — stop and report.

**Step 3 — Create and publish a test doc document:**
Click "Documentation Page" → "New document".
Fill in:
- Title: "Phase 2 Test — Doc"
- Slug: click "Generate" — confirm it auto-fills from the title
- Target Audience: select "All"
- Category: "test"
- Body: add one paragraph block with the text "Phase 2 schema validation test"
Click "Publish". Expected: document saves without validation errors and status changes to "Published".

**Step 4 — Create test documents for the other 3 types:**
Repeat the create-and-publish flow for:
- Release Note: fill Title + Slug + Sprint ID (e.g. "2026-01-a") + Body paragraph
- Article: fill Title + Slug + Tags (add one tag) + Body paragraph
- Reference Page: fill Title + Slug + Body paragraph

All four should publish without errors. Slug should auto-generate in each.

**Step 5 — Set Cloudflare Pages environment variables:**
Navigate to: Cloudflare Dashboard → Pages → your nxgen-docs project → Settings → Environment variables.
Add the following variables (apply to all environments — Production + Preview):

| Variable Name | Value |
|---------------|-------|
| SANITY_PROJECT_ID | (your project ID — find it in Sanity dashboard → project settings, or from the sanity init output) |
| SANITY_DATASET | production |
| SANITY_API_TOKEN | (create a new token: Sanity dashboard → project → API → Tokens → Add API token, Role: Editor) |

Note: These are NOT the `SANITY_STUDIO_*` prefixed vars. The Studio uses `SANITY_STUDIO_PROJECT_ID` locally; Cloudflare Pages uses the unprefixed `SANITY_PROJECT_ID` for the Phase 3 plugin.

**Step 6 — Clean up test documents:**
Delete the 4 test documents created in steps 3-4 (click the three-dot menu → Delete). This keeps the dataset clean for real content entry.
