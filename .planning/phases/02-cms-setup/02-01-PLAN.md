  <action>
From the repo root (C:/Users/abdel/.gemini/antigravity/scratch/nxgen-docs), run:

```bash
# Check whether a Sanity project already exists on this account
npx sanity@latest projects list 2>/dev/null || echo "Not authenticated — proceed with init"

# Scaffold the studio (creates studio/ subdirectory, prompts for login if needed)
npx sanity@latest init \
  --create-project "NXGEN Docs" \
  --dataset production \
  --output-path ./studio \
  --template clean \
  -y
```

After `sanity init` completes, REPLACE the generated `studio/sanity.cli.ts` with the following content (do not keep the generated hardcoded projectId):

```typescript
// studio/sanity.cli.ts
import {defineCliConfig} from 'sanity/cli'

export default defineCliConfig({
  api: {
    projectId: process.env.SANITY_STUDIO_PROJECT_ID!,
    dataset: process.env.SANITY_STUDIO_DATASET ?? 'production',
  },
  studioHost: 'nxgen-docs',
})
```

REPLACE the generated `studio/sanity.config.ts` with:

```typescript
// studio/sanity.config.ts
import {defineConfig} from 'sanity'
import {structureTool} from 'sanity/structure'
import {visionTool} from '@sanity/vision'
import {codeInput} from '@sanity/code-input'
import {table} from '@sanity/table'
import {schemaTypes} from './schemaTypes'

export default defineConfig({
  name: 'nxgen-docs',
  title: 'NXGEN Docs',
  projectId: process.env.SANITY_STUDIO_PROJECT_ID!,
  dataset: process.env.SANITY_STUDIO_DATASET ?? 'production',
  plugins: [
    structureTool(),
    visionTool(),
    codeInput(),
    table(),
  ],
  schema: {
    types: schemaTypes,
  },
})
```

Install the two Portable Text plugins from inside the studio directory:

```bash
cd studio && npm install @sanity/code-input @sanity/table
```

Create a minimal barrel file `studio/schemaTypes/index.ts`:

```typescript
// studio/schemaTypes/index.ts
// Schema types — populated in Plan 02
export const schemaTypes: unknown[] = []
```

CRITICAL: The Sanity CLI generates `sanity.cli.ts` and `sanity.config.ts` with the projectId hardcoded as a string literal. The files above must replace those generated versions so the project ID is read from environment variables. After replacement, create a `.env` file at `studio/.env` (gitignored) containing the real project ID returned by `sanity init`:

```
SANITY_STUDIO_PROJECT_ID=<actual-project-id-from-sanity-init-output>
SANITY_STUDIO_DATASET=production
```

The projectId is printed in the `sanity init` output. If it was not captured, run: `cd studio && npx sanity projects list` to find it.

<task type="auto">
  <name>Task 2: Register Sanity MCP server in Claude Code</name>
  <files>(no files — MCP registration modifies Claude Code user config, not the repo)</files>
  <action>
Register the remote Sanity MCP server at user scope so it is available in all Claude Code sessions on this machine:

```bash
claude mcp add Sanity -t http https://mcp.sanity.io --scope user
```

If the above command reports "already exists", the MCP is already registered — skip re-registration.

After registration, verify the MCP is listed:

```bash
claude mcp list
```

The Sanity MCP uses OAuth — on the next Claude Code session start, a browser-based OAuth prompt will appear. Complete the OAuth flow to grant Claude access to the Sanity project.

Then deploy the (empty) schema manifest so the MCP server has a project context:

```bash
cd /c/Users/abdel/.gemini/antigravity/scratch/nxgen-docs/studio && SANITY_STUDIO_PROJECT_ID=$(grep SANITY_STUDIO_PROJECT_ID .env | cut -d= -f2) npx sanity schema deploy
```

Note: An empty schema manifest is acceptable at this stage. Re-deploying with full schemas happens at the end of Plan 02.

Anti-pattern to avoid: Do NOT install the deprecated `@sanity/mcp-server` npm package. The remote mcp.sanity.io server is the supported path as of late 2025.
