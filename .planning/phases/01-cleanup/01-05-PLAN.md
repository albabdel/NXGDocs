Common patterns:
- Link points to a deleted page (storyblok-example, storyblok-preview, any page deleted in Plan 01/02) → remove or update the link to point to the correct current page
- Link points to a sidebar entry for a docs path that no longer exists → update `sidebars.ts` or the relevant sidebars file

- Link is an internal `href` in a `.tsx` page file → update the href

- `src/pages/` internal links: Edit the `.tsx` file that contains the broken `href`
- MDX doc links (`[text](../path)`): Edit the `.mdx` file
- Sidebar config (`sidebars.ts`, `sidebars-*.ts`): Edit the sidebar file to remove or update the dead reference
- Navbar links in `docusaurus.config.ts`: Update the `href` or `to` value

After fixing each batch of broken links, run `npm run build 2>&1 | grep -i "broken\|warn"` to confirm the count is decreasing. Continue until the build log shows zero broken-link warnings.
