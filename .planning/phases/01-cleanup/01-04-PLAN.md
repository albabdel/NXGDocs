  <action>
Read the current custom.css (starting point is the line count from Plan 02's SUMMARY). The goal is to reach under 2,000 lines.

**Dead rule categories to remove (in order):**

1. **CMS editor UI rules** — any rules referencing:
   - `storyblok`, `.sb-`, `tiptap`, `.ProseMirror`, `monaco`, `.monaco-editor`
   - `hygraph`, `strapi`, `payload`, `tinacms`, `.tina-`, `netlify-identity`
   Search for these strings in custom.css and delete the entire rule block containing each match.

2. **Dead component rules that survived Plan 02** — check for any remaining rules referencing components deleted in Plans 01 and 02:
   - particle, language-switcher, translation-glow, translation-progress
   - pdf-export, scroll-indicator, typing-animation, before-after
   - animated-stats, image-gallery, homepage-features, role-landing
   Delete any surviving rule blocks.

3. **Duplicate declarations** — look for selectors that appear more than once with identical properties. Keep the last declaration (CSS cascade order), delete the earlier one.

4. **Over-specified dead selectors** — rules that reference element IDs or class names that no longer exist in any `.tsx`, `.mdx`, or `.html` file. Use a targeted search:
   ```bash
   # Extract all class names used in CSS
   grep -o '\.[a-zA-Z][a-zA-Z0-9_-]*' /c/Users/abdel/.gemini/antigravity/scratch/nxgen-docs/classic/src/css/custom.css | sort -u > /tmp/css-classes.txt
   # Cross-reference against src/ and docs/ usage (spot-check the ones that look component-specific)
   ```
   Delete rules only when the class name is clearly a dead component reference (e.g., `.storyblok-preview`, `.tiptap-toolbar`) — not generic utility classes.

5. **Media query deduplication** — if the same breakpoint (`@media (max-width: 996px)`) appears multiple times with small rule sets, consolidate them into a single block.

**Stopping condition:** Stop at under 2,000 lines OR when only genuinely-shared/ambiguous rules remain. Do not strip rules you are uncertain about — it is better to be at 2,100 lines than to break a live component. Record the final line count.

After editing, run the build:

cd /c/Users/abdel/.gemini/antigravity/scratch/nxgen-docs/classic && npm run build
