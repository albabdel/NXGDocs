## 01. Prepare Your Environment

GCXONE devices must communicate with cloud services for video streaming, alarm transmission, and time synchronization.

### Required Ports & Endpoints

| Purpose | Endpoint / Port | Protocol |
| :--- | :--- | :--- |
| Primary Gateway | `18.185.17.113` | TCP |
| NTP Server | `time1.nxgen.cloud:123` | UDP |
| Secure Dashboard | `*:443` | TCP (HTTPS) |
| Video Stream | `*:554` | TCP (RTSP) |

<Callout type="tip" title="Quick Reference">
  Ensure your firewall allows outbound traffic to <strong>*.nxgen.cloud</strong>. If strict whitelisting is required, see the full domain list below.

## What to Do Next

- **Harden access:** Finalize MFA or SSO policies and rotate initial credentials. See [Password Management](/docs/getting-started/password-management).
- **Reduce noise:** Tune smart event thresholds and schedules once the pilot runs for a day. See [IP Whitelisting](/docs/getting-started/ip-whitelisting) for stable connectivity.
- **Expand coverage:** Onboard remaining sites, set retention rules, and standardize naming.
- **Integrate workflows:** Connect notifications to ticketing or chat for faster response.
