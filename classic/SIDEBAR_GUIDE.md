# GCXONE Documentation - Sidebar Guide

## Overview

The sidebar is **manually configured** in `sidebars.ts`. This means when you add new files through the admin panel, they **won't automatically appear** in the sidebar unless you update the configuration.

## How Docusaurus Doc IDs Work

Docusaurus automatically strips number prefixes from folder and file names to create "doc IDs":

- File: `docs/01-platform-overview/01-introduction.md` → Doc ID: `platform-overview/introduction`
- File: `docs/features/arm-disarm.md` → Doc ID: `features/arm-disarm`
- File: `docs/devices/hikproconnect-setup.md` → Doc ID: `devices/hikproconnect-setup`

The number prefixes (like `01-`, `02-`) help organize files in your filesystem but are removed from URLs.

## How to Add a New Document to the Sidebar

### Step 1: Create or Edit the File

Use the admin panel at http://localhost:3000/editor to create/edit your markdown file.

**Example:** Creating a new device guide at `docs/devices/new-device-setup.md`

```markdown
---
title: New Device Setup Guide
description: Setup guide for integrating New Device with GCXONE
tags: [devices, integration, new-device]
---

# New Device Setup Guide

Your content here...
```

### Step 2: Update sidebars.ts

Open `sidebars.ts` and add the new entry to the appropriate section.

**Example:** Adding to the Device integrations section:

```typescript
{
  type: 'category',
  label: 'Device integrations',
  items: [
    { type: 'doc', id: 'devices/index', label: 'Overview' },
    // ... existing items ...

    // Add your new device here:
    { type: 'doc', id: 'devices/new-device-setup', label: 'New Device' },
  ],
},
```

### Step 3: Save and Reload

The Docusaurus dev server will automatically detect the change and rebuild. Refresh your browser to see the new sidebar item.

## Adding a New Feature

**File location:** `docs/features/my-new-feature.md`

**Add to sidebar:**

```typescript
{
  type: 'category',
  label: 'Features',
  items: [
    { type: 'doc', id: 'features/index', label: 'Features overview' },
    // Add here:
    { type: 'doc', id: 'features/my-new-feature', label: 'My New Feature' },
    // ... other features ...
  ],
},
```

## Creating a New Category

If you want to add a completely new section:

```typescript
{
  type: 'category',
  label: 'My New Section',
  link: {
    type: 'generated-index',
    title: 'My New Section',
    description: 'Description of this section',
  },
  collapsible: true,
  collapsed: true,  // Start collapsed
  items: [
    { type: 'doc', id: 'my-section/first-doc', label: 'First Document' },
    { type: 'doc', id: 'my-section/second-doc', label: 'Second Document' },
  ],
},
```

## Auto-Generated Sections

You can also use auto-generation for folders:

```typescript
{
  type: 'category',
  label: 'Release Notes',
  items: [
    { type: 'autogenerated', dirName: 'release-notes' }
  ],
},
```

This will automatically include ALL markdown files in the `docs/release-notes/` folder.

## Important Sidebar Syntax

```typescript
// Single document
{ type: 'doc', id: 'folder/filename', label: 'Display Name' }

// Category with nested items
{
  type: 'category',
  label: 'Category Name',
  items: [
    // ... nested items ...
  ],
}

// Auto-generate from folder
{ type: 'autogenerated', dirName: 'folder-name' }

// External link
{ type: 'link', label: 'External Link', href: 'https://example.com' }

// HTML/Custom element
{
  type: 'html',
  value: '<div>Custom HTML</div>',
  defaultStyle: true,
}
```

## Troubleshooting

### "Doc ID doesn't exist" Error

**Problem:** Sidebar references a doc that doesn't exist.

**Solution:**
1. Check if the file exists at `docs/folder/filename.md`
2. Verify the doc ID matches the file path (without number prefixes)
3. Remove or comment out the sidebar entry if the file is missing

### Document Not Appearing in Sidebar

**Problem:** You created a file but it's not in the sidebar.

**Solution:** The sidebar is manually configured. You must add the entry to `sidebars.ts`.

### "No docs found in folder" Warning

**Problem:** Warning like "No docs found in 'user-guides'"

**Solution:** Either:
- Remove the autogenerated entry from sidebars.ts
- Add markdown files to that folder

## Best Practices

1. **Organize by topic** - Group related docs together
2. **Use clear labels** - Make sidebar labels descriptive
3. **Keep categories collapsed** - Use `collapsed: true` for large sections
4. **Match file organization** - Sidebar order should match logical reading order
5. **Test locally** - Always test sidebar changes before committing
6. **Use comments** - Add comments in sidebars.ts to separate sections

## Admin Panel & Sidebar Workflow

When editing through the admin panel:

1. **Create/Edit Content** - Use http://localhost:3000/editor
2. **Update Sidebar** - Manually edit `sidebars.ts` in your code editor
3. **Test** - Reload the site and verify the sidebar shows your new doc
4. **Commit** - Commit both the markdown file AND sidebars.ts changes
