# ✉️ Inviting Users to GCXONE

Onboarding new team members is handled through an automated email workflow. This ensures that every user securely verifies their identity and sets their own private credentials.

## 🏗️ Step-by-Step Onboarding

<Steps>

### 1. Initiation
1. Navigate to **Settings** → **Users**.
2. Click **Invite New User**.

### 2. Enter Identity Details
- **Email:** This must be a unique, valid email address. It will serve as their login ID.
- **Personal Info:** Enter First Name, Last Name, and optionally a Phone Number for SMS alerts.

### 3. Configure Access
- **Role:** Select the [pre-defined role](/docs/getting-started/user-management/roles-and-access-levels) that matches their responsibilities.
- **Customer Group:** (Optional) Select a [Customer Group](/docs/getting-started/user-management/customer-groups) if you need to restrict their view to a specific client subset.
- **Session Timeout:** Set the inactivity limit (default 30 mins).

### 4. Verification Workflow
Once you click **Submit**, GCXONE sends two automated emails to the user:
1. **Confirmation Email:** Verifies the email is active.
2. **Password Setup Link:** Allows the user to create their secure password.

</Steps>

---

## 🌐 Multi-Tenant Access

If a user (such as a consultant) is invited to multiple different GCXONE organizations using the **same email address**:

- **Single Identity:** They only need one password.
- **Tenant Selection:** Upon login, they will be prompted to choose which organization they want to access for that session.
- **Switching:** They can switch between tenants anytime via **Settings** → **Switch Tenant**.

---

## 🛠️ Troubleshooting

<Callout type="warning" title="Invitation Links Expire">
For security, the password setup link expires after **24 hours**. If a user misses this window, you must **Resend Invitation** from the Users table.
</Callout>

- **Email Not Received:** Ask the user to check their Spam/Junk folder for mail from `@nxgen.cloud`.
- **Login Issues:** Ensure the user clicked the **Verification Link** in the first email before attempting to use the password setup link.
