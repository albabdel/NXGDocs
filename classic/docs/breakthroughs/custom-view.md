## See It in Action

Custom View transforms how every role interacts with security data across GCXONE deployments:

### Scenario 1: Security Operations Center Optimization

A monitoring center handles 50+ sites with 800+ cameras. Operators were losing 30 minutes per shift navigating between screens to check alarm trends, site status, and camera health. Each manual check interrupted their focus on live monitoring.

With Custom View:
1. The SOC manager creates a role-specific dashboard with 6 key widgets: Active Alarms, Site Health Summary, Top 10 Busiest Cameras, Today's Alarm Trend, Offline Devices, and Quick Site Access
2. Operators now see all critical metrics on a single screen within 5 seconds of login
3. When an alarm triggers, the dashboard automatically highlights the affected site and shows recent alarm history for context
4. Response accuracy improves by 40% because operators have complete situational awareness before acting

The result: Each operator saves 4 hours per day previously spent gathering context, and alarm triage becomes 3x faster with all relevant data visible at a glance.

### Scenario 2: Multi-Tenant Service Provider Reporting

An MSP manages security for 100+ client sites and needs to generate monthly performance reports showing uptime, incident counts, and response metrics. Previously, an administrator spent 6-8 hours manually pulling data from multiple sources and formatting spreadsheets.

With Custom View:
- The administrator creates customer-specific dashboards that auto-populate with KPIs: 99.8% Camera Uptime, 247 Incidents Handled, 2.3 Min Average Response Time
- Each dashboard is assigned to the corresponding customer group for instant access
- Clients can view their real-time metrics 24/7 without requesting reports
- Monthly reporting time drops from 8 hours to 15 minutes of dashboard review and PDF export

The measurable impact: 93% reduction in reporting workload, immediate client visibility into service quality, and elimination of manual data errors that previously caused billing disputes.

### Scenario 3: Executive Decision-Making Dashboard

A General Manager overseeing 12 regional sites needs to monitor operational efficiency but doesn't have time to learn complex security software. They need answers to: "Which sites are performing well? Where are we at risk? What's our overall health?"

With Custom View:
1. IT creates an executive dashboard with simplified, high-level widgets: Sites Online (12/12), This Month's Capacity (+23%), Critical Issues (0), Revenue Impact (+15%)
2. Each metric is color-coded: green for healthy, yellow for attention needed, red for critical
3. The dashboard loads in under 2 seconds and updates in real-time
4. The GM can drill down into any metric with one click to see site-specific details

The transformation: Leadership makes data-driven decisions without training, identifies growth opportunities 95% faster, and gains confidence in the system's ROI with quantified performance metrics visible at all times.

## ⚙️ How to Activate Custom View

This section explains how to enable and configure Custom View in your GCXONE instance.

### Prerequisites

Before activating Custom View, ensure you have:
- [ ] Administrator access to GCXONE
- [ ] Custom View license enabled
- [ ] Appropriate user permissions for dashboard configuration

### Activation Steps

#### Step 1: Access Feature Settings
1. Log into GCXONE as an administrator.
2. Navigate to **Settings** → **Features** → **Breakthrough Features**.
3. Locate **Custom View** in the features list.

#### Step 2: Enable the Feature
1. Click the **Enable** button next to Custom View.
2. Review the feature capabilities and data visualization options.
3. Click **Confirm Activation**.

#### Step 3: Configure Dashboard Widgets
1. Go to **Insights** → **Dashboards**.
2. Click **Add Widget** to select from the library of KPI and video widgets.
3. Drag and drop widgets to create your optimal layout.
4. Click **Save Layout**.

#### Step 4: Define Role-Based Views
1. Navigate to **Access Management** → **Roles**.
2. Select a role (e.g., Operator) and click **Dashboard Assignment**.
3. Assign the newly created Custom View as the default dashboard for that role.

#### Step 5: Verify Visibility
1. Log in as a user with a specific role.
2. Verify that the dashboard loads correctly with the assigned widgets.
3. Ensure real-time data is flowing into the KPIs.

## 🚀 Adoption Guide

Successfully rolling out Custom View across your organization requires planning and training. This guide helps you maximize adoption and ROI.

### Phase 1: Planning (Week 1)

#### Assess Your Needs
- **Inventory:** List the key performance indicators (KPIs) each team needs to track.
- **Timeline:** Define the rollout schedule for different departments.
- **Scope:** Start with high-impact roles (e.g., Security Operators, Site Managers).
- **Resources:** Designate a "Dashboard Architect" to design the primary views.

### Phase 2: Pilot Testing (Week 2)
- Roll out to a small group (5-10 users) of operators.
- Gather feedback on widget layout and data relevance.
- Refine the dashboard designs based on actual usage patterns.

### Phase 3: Team Training (Week 2-3)
- Host a "Data-Driven Decisions" webinar.
- Provide a "Widget Glossary" explaining each data point.
- Ensure all users know how to "Reset Layout" if they accidentally change their view.

### Phase 4: Full Rollout (Week 3-4)
- Deploy role-based dashboards to all users.
- Standardize reporting based on Custom View insights.
- Disable legacy reports to encourage full adoption.

### Phase 5: Optimization (Ongoing)
- Review dashboard usage quarterly to retire unused widgets.
- Add new data sources as they become available in GCXONE.
- Conduct advanced training on "Drill-Down" capabilities.

### Best Practices
- **DO** keep dashboards focused; less is often more.
- **DO** use consistent color coding for status indicators.
- **DO** test your dashboards on mobile devices.
- **DON'T** clutter a single view with more than 6-8 widgets.
- **DON'T** use highly sensitive data in shared public dashboards.
