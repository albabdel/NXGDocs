# How to Adjust User/Operator Privileges in Genesis

This guide explains how to adjust privileges for users and operators in the Genesis platform.

## Creating a New Role

To create a new role with specific privileges:

1. **Log in to the Genesis platform**
2. **Navigate to Talos Workflow**
3. **Go to the Company section**
4. **Click on Settings**
5. **Select Users**
6. **Add and save a Role** based on your requirements
7. **Create a User Type** for the role you just created

:::tip Detailed Role Configuration
For comprehensive role management including privileges, access levels, and customer groups, see the [User Management Guide](/docs/admin-guide/user-management).
:::

## Changing a User's Role

To change the role assigned to an existing user:

1. **Navigate to the Company section**
2. **Go to Users**
3. **Click on the desired user** from the list
4. In the user details, select the appropriate **Role** from the dropdown menu
5. **Click on Submit**

:::info Immediate Effect
Role changes take effect immediately. The user may need to refresh their browser or log out and log back in to see the updated permissions.
:::

## Understanding Privileges

Privileges in Genesis determine what actions users can perform. Common privilege categories include:

- **View**: Ability to see and access information
- **Create**: Ability to create new records or configurations
- **Edit**: Ability to modify existing records or configurations
- **Delete**: Ability to remove records or configurations

Privileges are typically organized by:
- **Application**: Which apps the user can access
- **Category**: Specific sections within applications
- **Action**: What operations can be performed
