# GCXONE vs Talos: How They Work Together

Evalink Talos is a specialized, automated, cloud-based platform for monitoring stations designed to manage security alarms. It enhances the efficiency of security operations through advanced alarm management. Key Aspects of Talos: Architecture: Talos is highly reliable, built using Amazon Web Services (AWS) servers across America, Europe, and Asia in different availability zones. It is composed of many small, independent microservices. Role in Alarm Flow: Alarms generated by devices often flow to GCXONE first, where they are processed with analytics, and the outcome ("real alarm" or "false alarm") is then sent back to Talos as a follow-up alarm for enriched and verified event data. Site Synchronization: When a site is created in GCXONE, the site name is automatically created in Talos. Site data synchronization has also been implemented. Alarm Definition: An alarm is defined as a single notification from a security device (e.g., motion detected by a camera) requiring attention. Interface Screens: Operators interact with Talos via three main screens, typically on multiple monitors: the Alarm Receiving Screen (displays incoming alarms, sortable by priority or time), the Video Screen (shows live or recorded video clips linked to alarms), and presumably a third dashboard/overview screen. Operator Roles: Roles are typically segregated: the Night Shift focuses on critical alarms (intrusion, fire), deferring technical issues. The Day Shift handles technical alarms, maintenance, workflow adjustments, and customer interaction.
