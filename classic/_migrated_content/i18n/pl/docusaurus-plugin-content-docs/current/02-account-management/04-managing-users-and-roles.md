# Managing Users and Roles

User management in GCXONE/Talos is controlled by roles and workspaces.
Roles: Roles define permissions based on APP, category, action, and API. Common roles include Admin, Customer Admin, Backend API QA, and Operator.
Workspaces: Workspaces route alarms and restrict which operators can handle certain alarms (e.g., Standard vs. VIP workspace). A user can be restricted to specific work groups, site groups, or site types.
Auto-feed: Alarms can be forced to be automatically assigned to a user (auto-feed) and prevented from being blocked, depending on customer preference.
