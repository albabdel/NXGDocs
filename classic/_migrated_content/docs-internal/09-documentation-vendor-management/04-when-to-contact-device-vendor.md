# When to Contact Device Vendor

Maintaining accurate and comprehensive documentation is a continuous process essential for feeding the "Genie" (the AI knowledge base).
New Findings and Custom Parameters: Any information regarding custom parameters or new solutions that is not currently present in the documentation must be added as documentation.
False Alarm Filtering Documentation: A separate article dedicated solely to false alarm filtering should be created. This article must detail the necessary configurations required for the false alarm filtering mechanism to be enabled at the device level.
Port Information Detail: Documentation must specify what will happen if a specific required port is not enabled.
AI Analytics List: A comprehensive list of all objects that both current AI versions (e.g., Nova and the four identifiable objects) can detect must be compiled and documented.
