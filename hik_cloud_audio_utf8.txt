Important Note:┬á┬á
Both URLs must point to the correct domain depending on the mode (Cloud or Local).
Incorrect configuration may result in failure of cloud audio functionalities.
These URLs can be configured either at the Service Provider level or at the Device level.
1. If configured at the Service Provider level, the settings will be applied to all Hikvision devices
under that service provider.
2. Or, you can set the URLs separately for each device if you need different settings for different devices
or modes.
If you want to use the Device level configuration, you must remove the URLs from the Service Provider
level.┬áOtherwise, the system will always use the URLs defined in the Service Provider configuration, and
the Device level configuration will be ignored.

Γ₧ö Update this field with the latest┬áNode-RED version number┬áthat cloud audio supported (i.e.,┬á2.8.2).
https://helpdesk.nxgen.io/portal/en/kb/articles/hikvision-cloud-audio-implementation
Important Reason:┬á┬á
The local mode Node-RED version installed on the system must be greater than or equal to the version
specified in the Cloud Audio Supported Version field.
If the installed version is lower than the specified version, the browser microphone will not work
properly when enabling audio in the Video Viewer page.
┬áAlso, if the Cloud Audio Supported checkbox is not enabled in the capabilities, the cloud audio
functionality will not work at all.
Video Viewer:┬á┬á
1. On the Video Viewer page, perform the following steps:
Drag and drop the desired video stream into the viewer area.
Enable the mic by selecting respective channel. (Refer to the Hikvision Audio Documentation for
channel information).
2. Once the audio is enabled:
The browser will prompt for microphone access, or the microphone icon in the browser toolbar will
indicate that the mic is active.
This confirms that Cloud Audio is enabled and the audio data is being transmitted to the cloud
service.
